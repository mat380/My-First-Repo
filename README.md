# Leroi Mega Center — Customer Analytics Project

Cannabis retail customer data pipeline: segment customers by recency using Cova POS export data, then drive re-engagement through SpringBig SMS campaigns.

---

## What We're Building Toward

The end goal is a closed-loop marketing system:

1. **Export** customer transaction history from **Cova POS**
2. **Segment** customers by how recently they've purchased (active / lapsed / lost)
3. **Prioritize** lapsed high-value customers for personal outreach or targeted campaigns
4. **Send** re-engagement SMS via **SpringBig Autoconnect**
5. **Measure** campaign ROI using SpringBig's performance reports
6. Repeat on a regular cadence (weekly or monthly)

## Project Files

### Scripts

#### `segment_customers.py`
Reads a customer CSV and splits it into three segments based on days since last purchase.

**How to run:**
```bash
python segment_customers.py                   # uses customers.csv by default
python segment_customers.py customers_from_history.csv   # specify a different file
```

**Input file requirements** — CSV with these exact column headers:
| Column | Format | Example |
|---|---|---|
| `first_name` | text | Alice |
| `last_name` | text | Johnson |
| `last_purchase_date` | YYYY-MM-DD, MM/DD/YYYY, or DD-MM-YYYY | 2026-03-10 |
| `total_spend` | number | 245.00 |
| `loyalty_points` | number | 1200 |

**Segmentation logic:**
| Segment | Threshold | Output file |
|---|---|---|
| Active | Purchased within last 30 days | `customers_active.csv` |
| Lapsed | Purchased 31–60 days ago | `customers_lapsed.csv` |
| Lost | Last purchase was 61+ days ago | `customers_lost.csv` |

Rows with unparseable dates are skipped with a warning. The script prints a summary count when done.

### Data Files

#### Input

| File | Source | Description |
|---|---|---|
| `customers.csv` | Manual / test data | Small sample file (8 customers) used for testing the script |
| `customers_from_history.csv` | **Cova POS export** | Full customer history (~51,792 rows). Exported from Cova's reporting module. Contains messy real-world names (special characters, nicknames, notes embedded in name fields). |

**Note on `customers_from_history.csv`:** This is the real dataset. Some rows have blank names, encoded discounts in the name field (e.g., `-10% OFF First Nations`), or other data quality issues from how customers were entered in the POS. The segmentation script handles missing dates gracefully but name cleanup may be needed before using this data for outreach.

|---|---|---|
| `customers_active.csv` | ~4,091 | Purchased within the last 30 days |

## Cova POS — Getting the Customer Export

1. Log into Cova back office
2. Navigate to **Reports → Customer Report** (or Customer History)
3. Export as CSV
4. Save as `customers_from_history.csv` in this project folder
5. Run `segment_customers.py customers_from_history.csv`

The export should include at minimum: customer name, last purchase date, total lifetime spend, and loyalty points.

---

## SpringBig — Autoconnect Campaigns

SpringBig's Autoconnect feature sends automated SMS messages triggered by customer behavior. Current campaigns include:

- **$1000+ Purchase** — triggers after a large transaction
- **Customer Join** — welcome message for new loyalty members

To pull reports: SpringBig dashboard → **Reports → Autoconnect** → Export CSV or XLSX.
